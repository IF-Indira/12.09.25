БЛОК 3: ПРЕДОБРАБОТКА ДАННЫХ
One-Hot Encoding для 'brand':
Было: ['Honda', 'Yamaha', ...]
Стало: brand_Honda: [1, 0, 0, 0, 0], brand_Yamaha: [0, 1, 0, 0, 0], ...
drop='first' удаляет первый столбец для избежания мультиколлинеарности
StandardScaler для числовых признаков:
Формула: (x - mean) / std
Все признаки приводятся к среднему=0, стандартное отклонение=1
Ускоряет обучение нейросети
Результат: Матрица X_processed готова для подачи в нейросеть

БЛОК 4: СОЗДАНИЕ МОДЕЛИ НЕЙРОННОЙ СЕТИ
Архитектурные решения:
1.	Полносвязная сеть (FCN): Каждый нейрон связан со всеми нейронами следующего слоя
2.	Активация ReLU: f(x) = max(0, x) - ускоряет обучение, решает проблему затухающих градиентов
3.	Dropout: Случайное отключение нейронов для предотвращения переобучения
4.	Выходной слой: 1 нейрон с линейной активацией для регрессии
5.	Оптимизатор Adam: Адаптивная скорость обучения

Блок 5: КРОСС-ВАЛИДАЦИЯ
Процесс кросс-валидации:
1.	KFold разделяет данные на 5 частей
o	Итерация 1: Части 2-5 → обучение, часть 1 → валидация
o	Итерация 2: Части 1,3-5 → обучение, часть 2 → валидация
o	... и т.д.
2.	EarlyStopping: Останавливает обучение, если 10 эпох подряд нет улучшения на валидации
3.	Цель: Оценить стабильность модели на разных подвыборках

Блок 6: ВИЗУАЛИЗАЦИЯ РЕЗУЛЬТАТОВ
Что показывает каждый график:
1.	Loss: Сходится ли модель, нет ли переобучения
2.	MAE: Точность в абсолютных значениях
3.	Scatter: Насколько близки предсказания к реальности
4.	Histogram: Распределение ошибок (должно быть нормальным)
5.	Feature Importance: Какие признаки важнее для модели
6.	Residuals: Есть ли паттерны в ошибках
